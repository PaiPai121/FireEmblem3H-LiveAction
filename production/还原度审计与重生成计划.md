# 还原度审计与重生成计划

日期：2026-07-04

## 目标

把真人化候选从“气质相近”推进到“妆造、服饰、角色识别点可被游戏玩家认出”。

本轮先处理最高优先级角色：

- Byleth
- Sothis
- Edelgard
- Dimitri
- Claude
- Rhea

## 版权处理

公开仓库和 GitHub Pages 不建议直接批量存放官方设定图、立绘、截图。

推荐做法：

- 在 `references/official_sources.md` 记录官方/百科参考链接。
- 在 `fidelity_review.md` 拆解视觉锚点和当前候选问题。
- 本地如需下载官方参考图，可放入 `references/official_local/`，并避免推送到公开仓库。
- 生成图可以公开检阅，但提示词和审片结论必须保留来源意识。

## 文件规范

每个角色优先补齐：

```text
<Character>/
  references/
    official_sources.md
  fidelity_review.md
  <era>/
    regeneration_prompt.md
```

## 审计维度

- Hair：发色、发型、刘海、长度、标志性轮廓。
- Face：年龄感、眼色、表情、气质。
- Costume：主色、剪裁、领口、披风、纹章、装饰、材质。
- Silhouette：远看是否像该角色。
- Faction：阵营颜色和符号是否明确。
- Avoid：最容易跑偏的生成问题。

## 生成原则

- 不写“cosplay”，写“live-action adaptation with faithful costume silhouette”。
- 不只写角色名，必须把官方视觉锚点拆成具体词。
- 学院篇要保留 15-18 岁学生感，不要做成成熟君主或战争指挥官。
- 服装要优先还原轮廓和色块，再追求复杂纹样。
- 每次重生成先出 4 张候选和 1 张 contact sheet，再人工审片，不直接覆盖 `selected.png`。

## 本轮结论

当前候选整体电影感足够，但大多是“阵营配色 + 现实制服”的近似方案。下一轮应该重点强化：

- Byleth：官方黑色长外套、内层护甲/纹样、青绿色发色和冷淡佣兵感。
- Sothis：头饰、侧发团、尖耳、蓝紫金服饰和更古老的神性。
- Edelgard：学院制服的红白黑结构、短银发轮廓、帝国符号。
- Dimitri：蓝狮学院制服的白蓝黑结构、王子感和更明确的金发轮廓。
- Claude：黄色金鹿披风、弓手/战术家身份、棕发绿眼和狡黠感。
- Rhea：大主教白金绿礼服、头饰/额饰、神圣压迫感。
